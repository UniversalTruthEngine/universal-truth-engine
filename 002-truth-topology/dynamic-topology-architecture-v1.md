# Dynamic Topology Architecture v1

## Purpose

This document defines the first architecture for making the Universal Truth Engine topology metadata-driven rather than manually curated.

The long-term goal is that the 3D truth map should be generated from Core Truth metadata.

---

## Principle

The map should not be the source of truth.

The source of truth should be the Core Truth metadata.

The map should derive:

- nodes,
- edges,
- dependency centrality,
- proof links,
- domains,
- confidence levels,
- regions,
- and traversal paths

from structured metadata.

---

## Current Problem

Early UTE map versions manually define much of the graph data.

That works for prototypes, but it will not scale to hundreds or thousands of Core Truths.

---

```text
001-fact-vault/

3D Map Engine
```

---

## Generated Node Fields

Each map node should be generated from metadata and include:

- id
- title
- domain
- confidence_level
- dependencies
- proof_file
- entry_file
- dependency_centrality
- recursive_dependency_count
- depended_on_by
- bridge_score
- stable_coordinates if available

---

## Generated Edge Fields

Each edge should include:

- source
- target
- type
- weight
- direct_or_recursive
- description

Default direct dependency type:

```text
depends_on
```

---

## Derived Metrics

### Dependency Centrality

How many other truths directly depend on this truth.

### Recursive Dependency Centrality

How many truths depend on this truth directly or indirectly.

### Bridge Score

A provisional measure of whether a truth connects multiple domains.

### Proof Completeness

A future measure based on whether the truth has:

- entry file,
- proof file,
- metadata file,
- validation layer,
- reconstruction method,
- failure cases.

---

## Stable vs Dynamic Coordinates

Stable coordinates are navigational addresses.

Dynamic coordinates may be generated by layout algorithms.

The dynamic topology engine may propose coordinates, but stable coordinate changes require topology review.

---

## Future Capabilities

The dynamic topology system should eventually support:

- missing dependency detection,
- circular dependency detection,
- high-centrality weak-proof detection,
- isolated-node detection,
- bridge-node analysis,
- domain clustering,
- reconstruction pathfinding,
- topology health reports.

---

## Final Principle

UTE should eventually generate its visible universe from its truth objects.

The map should emerge from the archive.
